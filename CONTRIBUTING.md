# Участие в разработке

1. Создайте ветку от `main`.
2. Не добавляйте потоки, credentials или EPG без первичного подтверждения прав.
3. Не расширяйте реестр за пределы 28 записей без отдельного решения владельца проекта.
4. Добавьте тест на изменённое поведение.
5. Выполните `npm run validate`.
6. Опишите платформу и результат ручного smoke test.

Pull request с токеном, cookie, паролем, подписанным URL, пиратским агрегатором или обходом DRM
должен быть закрыт и секрет — отозван.

Код форматируется Prettier. Пользовательские строки вставляются через безопасные DOM API.
Зависимости production runtime требуют отдельного обоснования; предпочтителен vanilla TypeScript.
