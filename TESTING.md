# Тестирование

## Команды

```powershell
npm run lint
npm run typecheck
npm test
npm run build
npm run check:size
npm run check:security
npm run validate
```

`validate` выполняет весь обязательный набор последовательно. Тесты не делают сетевых запросов и
не используют реальные телеканалы.

## Покрытые сценарии

- BOM, CRLF, кавычки, запятые, `EXTGRP`, `EXTVLCOPT`, pipe headers и относительные M3U URL;
- повреждённые записи и запрет активных/локальных URL-схем;
- ровно 28 уникальных записей и неизменный порядок нумерации;
- точное и alias-сопоставление, ручной mapping и игнорирование посторонних каналов;
- миграции, пределы настроек, фильтрация неизвестных ID, избранное и история;
- XMLTV timezone, программа/прогресс и защита от DTD/entity;
- маскирование URL/заголовков и экспорт без секретов;
- интеграционный путь M3U → runtime-каталог → Lampa Player с резервным URL.

## Браузерное демо

После `npm run build`:

```powershell
npx http-server site -p 4173 -c-1
```

Откройте `http://127.0.0.1:4173/demo/`. Проверьте:

1. в меню появился единственный пункт «ТВ»;
2. открывается категория «Основные» с 13 каналами;
3. карточка без URL открывает редактор источника;
4. стрелки, Enter и Escape меняют фокус/Activity;
5. раздел 18+ требует установки PIN;
6. после перезагрузки повторная загрузка не дублирует меню;
7. консоль браузера не содержит необработанных ошибок.

Демо намеренно не запускает сеть и не отображает взрослые карточки.

## Реальное устройство

Для smoke test требуются законный тестовый M3U и доступ к Lampa. Минимальная матрица:

- Android TV / Android WebView;
- desktop Chromium;
- Tizen, если доступен;
- webOS, если доступен.

Проверяются установка по HTTPS, пульт, внутренний/внешний плеер, CORS-классификация, XMLTV,
резервный источник и удаление. Отсутствующее физическое устройство нельзя заменять заявлением об
успешном тесте.
