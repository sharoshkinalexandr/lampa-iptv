# Безопасность

## Поддерживаемая версия

Исправления безопасности выпускаются для последней стабильной версии.

## Сообщение об уязвимости

Не публикуйте рабочие IPTV credentials, cookies, токены или подписанные URL в issue. Используйте
приватный канал GitHub Security Advisory репозитория после публикации. В отчёте укажите версию,
платформу, минимальные шаги и обезличенный пример.

## Модель угроз

- M3U/XMLTV/backup считаются недоверенными данными;
- UI использует `textContent`, не вставляет пользовательский HTML;
- запрещены `eval`, `new Function`, `javascript:`, `data:`, `file:` и `vbscript:`;
- XMLTV с DTD/entity отклоняется;
- действуют лимиты размера, количества записей, программ, истории и резервов;
- storage миграция удаляет неизвестные channel ID и опасные URL;
- логи и обычный backup маскируют secrets;
- Actions читают только versioned allowlist `public/data/sources.json`, не пользовательское
  хранилище.

## Credentials

Xtream login/password, Authorization, Cookie, Referer и URL с токенами хранятся только локально.
Они могут быть раскрыты пользователем через отдельный экспорт с секретами; такой файл следует
считать паролем и не прикладывать к issues.

PIN 18+ — локальная защита интерфейса, а не криптографическое хранилище. Хранится salted hash, но
владелец устройства с доступом к storage может сбросить состояние.

## Не поддерживается

Проект не обходит DRM, авторизацию, геоблокировки, CAPTCHA, Cloudflare или ограничения браузера.
Обнаруженный защищённый источник должен открываться официальным приложением/плеером провайдера.
